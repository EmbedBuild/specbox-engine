# Contributing to SpecBox Engine

## Before you start

This project is **source-available, not open source**. You are welcome to study the code, learn from it, and propose improvements. But access to this repository does not grant rights to use SpecBox Engine commercially or in production. See [LICENSE](LICENSE) and [COMMERCIAL_LICENSE.md](COMMERCIAL_LICENSE.md).

## Contributor License Agreement (CLA)

By submitting a Pull Request, you agree that:

1. Your contribution is your original work, or you have the right to submit it.
2. You grant Jesus Perez Sanchez (Embed Build) a perpetual, irrevocable, worldwide, royalty-free license to use, modify, and distribute your contribution as part of SpecBox Engine under any license.
3. You understand that your contribution will be subject to the project's BSL 1.1 license.
4. You are not entitled to compensation for your contribution unless a separate written agreement exists.

## Rules

1. **No direct pushes to `main`** — All changes must go through a Pull Request.
2. **Review required** — Every PR requires approval from the project owner (@jesusperezdeveloper).
3. **One feature per PR** — Keep PRs focused and small.
4. **No force pushes** — Linear history is enforced; rebase your branch before opening the PR.
5. **Resolve conversations** — All review comments must be resolved before merge.

## Protected files

The following files require explicit owner approval and should not be modified without prior discussion:

- `ENGINE_VERSION.yaml`
- `CLAUDE.md`
- `install.sh`
- `.claude/settings.json`
- `rules/GLOBAL_RULES.md`
- `server/server.py`
- `LICENSE`
- `COMMERCIAL_LICENSE.md`

## How to contribute

1. Fork the repository
2. Create a feature branch from `main`
3. Make your changes
4. Open a Pull Request using the provided template
5. Wait for review and address any feedback

## Philosophy

I share this project because I believe in transparency and building trust in the developer community. Contributions that improve SpecBox Engine are welcome and appreciated. But this is a professional tool that sustains a business — if you want to use it commercially, let's talk: **jesus@embed.build**
